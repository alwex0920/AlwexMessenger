# 🚀 AlwexMessenger

> Личный мессенджер с end-to-end шифрованием, созданный с нуля  
> Работает на бесплатной инфраструктуре (Cloudflare, Supabase, Vercel)

---

## ✨ Возможности

- 💬 Личные чаты (Direct messages)
- 👥 Группы с шифрованием
- 📢 Каналы (без шифрования)
- 🔐 End-to-End шифрование (AES-GCM + RSA)
- ✍️ Подпись сообщений (RSA-PSS)
- 📁 Отправка файлов и изображений (до 15MB)
- 🖼️ Превью изображений и видео
- 🔔 Уведомления (Web Push)
- 📱 Web + Android приложение
- ☎️ Звонки (через Cloudflare Realtime)
- 🔑 Импорт / экспорт ключей (QR + код)
- 🔄 Ротация ключей
- 🌍 Работа через прокси (обход блокировок)

## 🔐 Как работает шифрование

Используется гибридная схема:

- RSA — для передачи ключей  
- AES-GCM — для шифрования сообщений
- RSA-PSS — для подписи сообщений

### Пример:

1. Генерируется случайный AES-ключ  
2. Сообщение шифруется AES  
3. AES-ключ шифруется публичным ключом получателя (RSA)  
4. Сервер хранит только:
   - encryptedContent
   - encryptedKey
   - nonce
   - signature

Сервер не может прочитать сообщения.

---

## 🔐 Подпись сообщений

Каждое зашифрованное сообщение дополнительно **подписывается** отправителем с помощью его приватного ключа. Это даёт две гарантии:

- **Авторство** – сообщение действительно отправил владелец ключа, а не кто‑то другой.
- **Целостность** – содержимое не было изменено после отправки (даже самим сервером).

### Как это работает

1. Отправитель вычисляет цифровую подпись для `encryptedContent`, используя **RSA-PSS** и свой приватный ключ.
2. Подпись (`signature`) отправляется вместе с зашифрованным сообщением.
3. Получатель проверяет подпись **публичным ключом отправителя** сразу после расшифровки.
4. Если подпись не совпадает или отсутствует, под сообщением появляется предупреждение:
   - «⚠️ Подпись недействительна! Сообщение могло быть подделано.»
   - «⚠️ Сообщение не подписано»

### Что хранит сервер

Сервер **никогда не видит** приватный ключ и не может подделать подпись. В базу данных попадают только:

- `encryptedContent`
- `encryptedKey`
- `nonce`
- `signature`

Без подписи сервер (или злоумышленник, получивший доступ к базе) мог бы подменить зашифрованное содержимое, и получатель не заметил бы подмены. Подпись полностью исключает такую атаку.

### Проверка в клиенте

В интерфейсе под каждым входящим сообщением отображается статус подписи:

- ✅ Действительная подпись — сообщение помечается значком E2E (зелёный щит).
- ⚠️ Недействительная подпись — появляется предупреждение.
- ⚠️ Отсутствует подпись — также предупреждение (для старых сообщений или сообщений из каналов).
